ProjectFR 프로젝트(`/mnt/e/Agent/workspaces/Godot/projects/ProjectFR`)에 대해 작업 계획을 수립한다.

## 작업 요청

$ARGUMENTS

## 절차

`$ARGUMENTS`가 비어 있으면 **자동 스캔 모드**, 내용이 있으면 **요청 기반 모드**로 동작한다.

### 자동 스캔 모드 ($ARGUMENTS 없음)

1. `CLAUDE.md`, `AGENTS.md`, `README.md`, `docs/` 하위 파일들을 읽어 프로젝트 범위와 현재 상태를 파악한다.
2. 프로젝트 소스를 검사해 미완료 작업, 알려진 버그, 누락된 기능, 유용한 개선사항을 찾는다.
3. `docs/TODO.md`를 작성(또는 덮어쓰기)한다. 5~10개 항목, 우선순위 순 정렬.

### 요청 기반 모드 ($ARGUMENTS 있음)

1. `docs/TODO.md`를 읽고 기존 섹션 구조를 파악한다.
2. 위 작업 요청을 분석해 구현 항목으로 쪼갠다.
3. 기존 섹션에 맞게 항목을 추가한다. 새 섹션 필요 시 `## <섹션명>` 헤더와 함께 추가.

### 공통 규칙

- 각 항목은 `- [ ] <구체적인 구현 지시>` 형식, 항목 하나씩 독립 실행 가능하게
- 완료된 항목은 `- [x]` 형식
- `docs/TODO.md` 상단의 `> Updated:` 날짜를 오늘 날짜로 갱신
- 소스 코드를 실제로 수정하지 않는다
- 추가한 항목 목록을 간결하게 보고하고, `/todo`로 실행할 수 있음을 안내한다
