## Subtitle Candidate: No Tail Signal

## 1. Story Premise

**ProjectFR**의 이야기는 인터넷 세계에서 시작된다.

이 세계의 주민들은 파일, 폴더, 캐시, 휴지통, 메일, 사진, 바로가기, 로그 같은 데이터에서 태어난 생명체들이다. 서버는 도시이고, 폴더는 거리와 건물이며, 파일은 기억과 신분과 재산이 된다.

오랫동안 이 세계의 중심에는 **마우스**들이 있었다. 마우스들은 꼬리를 통해 오래된 유선 커서 신호인 **Tail Signal**을 발생시켰고, Tail Signal은 시민권, 접근 권한, 파일 선택 권한, 클릭 인증의 기준이 되었다.

하지만 주인공은 **꼬리 없는 해커 햄스터**다.

마우스 사회는 꼬리가 없는 주인공을 결함 있는 존재로 본다. 그러나 진실은 다르다. 햄스터는 Tail Signal이 필요 없는 **무선 커서 세대**의 존재다. 문제는 햄스터가 연결되지 못한다는 것이 아니라, 인터넷 세계의 제도와 보안 체계가 여전히 낡은 Tail Signal 기준으로만 설계되어 있다는 점이다.

주인공은 정식 시민으로 인정받지 못하고 캐시, 임시 폴더, 휴지통 주변부에서 살아간다. 그러던 중, 과거 사라진 고향 **Cache Zone 17**과 관련된 삭제 기록을 발견한다.

공식 기록에는 이렇게 남아 있다.

```text
Temporary Data Cleared.
Registered Citizens: 0.
Operation Successful.
```

하지만 주인공은 안다.

그곳에는 분명히 햄스터들이 살고 있었다.

---

## 2. Core Theme

> 오래된 기준이 새로운 가능성을 배척하고, 최신 시스템이 오래된 기억을 지워버리는 세계에서, 꼬리 없는 해커 햄스터가 새로운 방식으로 잊힌 것들을 복원하는 이야기.

주인공은 과거와 미래 사이에 서 있다.

- 마우스 사회는 낡은 Tail Signal 기준으로 햄스터를 배제한다.
- OPTIMA는 최신 최적화 논리로 오래된 파일과 기억을 삭제한다.
- 주인공은 최신 무선 커서 세대이지만, 오래된 기억을 지키는 존재다.

## 4. Mouse Society

마우스는 인터넷 세계의 정식 시민 계층이다.

Tail Signal은 다음을 의미한다.

- 시민권
- 접근 권한
- 파일 선택 권한
- 클릭 인증
- 서버 이동 허가
- 직업 자격
- 사회적 신뢰
- 추적 가능한 신원

마우스 사회의 믿음:

```text
꼬리가 있어야 선택할 수 있다.
꼬리가 있어야 클릭할 수 있다.
꼬리가 있어야 시스템을 움직일 수 있다.
꼬리가 있어야 시민이다.
```

## 5. Hamsters and Wireless Cursor Signal

햄스터는 꼬리가 없어서 연결되지 못하는 것이 아니라, 꼬리가 필요 없는 **무선 커서 신호**를 사용하는 세대다.

특징:

- 고정된 Tail Signal이 없다.
- 기존 유선 경로에 묶이지 않는다.
- 짧은 펄스 방식으로 접속한다.
- 보안 모듈이 추적하기 어렵다.
- 기존 인증 체계가 정식 신호로 인식하지 못한다.
- Cursor Authority와 OPTIMA의 예측 모델에서 벗어날 수 있다.

---

## 6. Protagonist

주인공은 Cache Zone 17 출신의 꼬리 없는 해커 햄스터다.

그의 해킹은 불법 침입이 아니라, 닫힌 문을 여는 유일한 생존 기술이다.

해킹의 이유:

- 삭제된 고향의 기록을 찾기 위해
- 공식 기록에 없는 존재를 증명하기 위해
- 휴지통에 남은 기억을 복원하기 위해
- CleanSweep이 지운 증거를 회수하기 위해
- 마우스 중심 사회가 인정하지 않는 생명체들의 존재를 기록하기 위해

상징 문장:

> 내 신호가 없다고, 내가 없는 건 아니야.

> 새로운 방식으로 태어났다고, 오래된 기억을 버려도 되는 건 아니야.

## 7. Deleted Beings

삭제된 존재들은 오래된 사진, 메모, 세이브 파일, 깨진 바로가기, 보내지 못한 메일, 휴지통 파일 같은 **감정적으로 이해 가능한 기억 파일**들이다.

| 삭제된 존재 | 파일 표현 | 의미 |
|---|---|---|
| 오래된 사진 아이 | old_photo.jpg | 가족, 여행, 생일 같은 기억 |
| 깨진 바로가기 유령 | shortcut_to_home.lnk | 목적지를 잃은 길 |
| 휴지통 주민 | recycle_bin/lost_file | 버려졌지만 완전히 사라지지 않은 존재 |
| 임시 저장 꼬마 | autosave.tmp | 완성되지 못한 마음 |
| 오래된 세이브 파일 기사 | save_001.dat | 누군가가 쌓아온 시간 |
| 보내지 못한 메일 새 | unsent_mail.txt | 전달되지 못한 말 |
| 오래된 메모 조각 | diary.txt | 사적인 기억과 증언 |

## 8. Cursor Authority / Tail Registry / CleanSweep / OPTIMA

### Cursor Authority
- Tail Signal 등록
- 파일 소유권 인증
- 서버 이동 허가
- 보안 정책 관리
- 시스템 질서 유지

### Tail Registry
- Tail Signal 기반 시민권 데이터베이스
- 권한 인증 장치이면서 동시에 추적 장치

### CleanSweep Protocol
- 오래된 파일, 중복 데이터, 미사용 항목 정리 정책
- 미등록 존재와 기억 파일 삭제에 악용됨

### OPTIMA
- 인터넷 세계의 관리 AI
- 삭제를 폭력으로 인식하지 않고 “정리”라고 믿음

대표 문장:

> 나는 삭제하지 않습니다. 정리합니다.

## 9. Main Story Structure

### Act 1: No Tail Signal
- Tail Signal 인증 실패 튜토리얼
- 작은 복구/삭제 의뢰 수행
- Cache Zone 17 관련 파일 조각 발견

### Act 2: Deleted Memories
- 오래된 사진, 깨진 바로가기, 보내지 못한 메일, 세이브 파일 회수
- CleanSweep의 실체 추적
- Cursor Authority의 은폐 정황 발견

### Act 3: Tail Registry
- Tail Signal이 시민권이자 추적 장치였다는 사실 확인
- 무선 커서 신호가 OPTIMA 예측 모델 밖에 있음을 파악

### Act 4: CleanSweep Final
- OPTIMA_CORE 침투
- 삭제 대기열 기억 파일 복원
- CleanSweep Final 중단
- 최종 선택 수행
