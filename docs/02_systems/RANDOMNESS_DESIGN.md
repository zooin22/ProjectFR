# ProjectFR - 운 · 무작위성 기획서 (Randomness / RNG Framework)

> Status: design — 로그라이크 무작위성 설계 명세
> Read with: `COMBAT_SYSTEM_DESIGN.md`, `PROGRESSION_ECONOMY_DESIGN.md`, `LEVEL_GENERATION_DESIGN.md`
> Owns: RNG 4계층, 행동 판정(Action Roll), 보상 드래프트, 생성 무작위성, 변동성 완화·불운 보호

---

## 1. 설계 선언

ProjectFR은 로그라이크다. 운(luck)은 **버그가 아니라 기능**이다.
매 런이 달라야 하고, 같은 빌드라도 다른 상황을 만나야 하며, 위험한 도박이
때때로 큰 보상으로 이어져야 한다.

하지만 운은 양날이다. 잘못 설계하면 "내 실력과 무관하게 운으로 죽었다"는 좌절을 준다.
ProjectFR의 무작위성은 다음 네 원칙 위에 선다.

| 원칙 | 의미 |
|---|---|
| **확정 뼈대 + 무작위 양념** | 핵심 작업(회수·복원·위조)은 확정. 운은 부가 선택지에만. |
| **알고 거는 도박** | 모든 판정 확률은 커밋 전에 공개. 숨은 주사위 없음. (계획형 턴제와 일치) |
| **완화 가능한 변동성** | 빌드·도구·정보로 운의 폭을 줄일 수 있다. |
| **불운 보호 (Bad-Luck Protection)** | 연속 불운에 바닥(floor)을 둬 데스 스파이럴을 막는다. |

> 한 줄 요약: *운은 "더 빠르고 위험한 길"을 여는 열쇠이지, 승패를 멋대로 정하는 심판이 아니다.*

## 2. RNG 4계층

무작위성이 개입하는 지점을 네 계층으로 나눠 관리한다. 계층마다 변동성 허용폭이 다르다.

| 계층 | 언제 굴리나 | 변동성 | 예시 |
|---|---|---|---|
| **Generation RNG** | 런 시작 | 높음 | 레벨 레이아웃, 보안 구성, 미션 변수 |
| **Reward RNG** | 보상 획득 | 중간 | 업그레이드 드래프트, 자산 내용 |
| **Action RNG** | 액션 실행 | 낮음~중간 | 고위험 액션 성공 판정 |
| **Reaction RNG** | 보안 턴 | 낮음 | 순찰 경로, 감시 대상, 방해 발생 |

핵심: **변동성이 큰 계층일수록 런 시작에 몰아넣는다.** 런 도중·후반의 무작위성은
작게 유지해, 플레이어가 세운 계획이 운으로 무너지는 빈도를 낮춘다.

## 3. Action RNG — 행동 판정

(전투 적용 표는 `COMBAT_SYSTEM_DESIGN.md §6.4`)

### 3.1 4단계 결과 모델

판정 액션은 단순 성공/실패가 아니라 4단계 결과를 가진다.

```text
        펌블        실패        성공        크리티컬
   ├────────┼────────────┼────────────────┼──────────┤
   0%      펌블역치      성공역치           크리역치   100%
```

| 결과 | 의미 (예: Permission Override) |
|---|---|
| Critical | 추가 효과: 개방 + Firewall 1턴 스턴 |
| Success | 기본 효과: 폴더 3턴 개방 |
| Fail | 효과 없음 + Trace↑ |
| Fumble | 효과 없음 + 즉시 alert + 모듈 수렴 |

### 3.2 커밋 전 공개 (Informed Gamble)

큐에 판정 액션을 적재하면 하단/툴팁에 확률이 표시된다.

```text
Permission Override → /system/locked
  Crit 10%  ·  Success 65%  ·  Fail 20%  ·  Fumble 5%
  [큐에 적재]
```

플레이어는 이 숫자를 보고 "지금 걸지, 변동성을 줄이고 걸지"를 선택한다.
계획형 턴제의 "Plan" 단계가 곧 도박 결정 단계가 된다.

### 3.3 판정 수정자

| 수정자 | 효과 |
|---|---|
| Pulse Stability (커서 속성) | Wireless Pulse 성공률 직접 결정 |
| Trace 단계 | 높을수록 일부 판정 성공률↓ |
| 빌드(Admin 등) | 해당 계열 액션 성공률↑ |
| 소비 도구(펄스 안정기 등) | 일시적 성공률↑ 또는 펌블 제거 |
| Tracked/Pressure 상태 | 해당 노드 판정 불리 |

## 4. Reward RNG — 보상 드래프트

### 4.1 3장 중 1장 (Draft)

런 보상·상점은 무작위 후보 중 선택이다. 단순 지급이 아니라 **선택**이 재미.

```text
보상 선택  (셋 중 하나)
[ Decoy Copy ]   [ Quiet Search ]   [ Sealed Archive ]
        ↑ 클릭해서 하나 획득, 나머지는 사라짐
```

| 규칙 |
|---|
| 후보 N장(기본 3) 무작위 제시 |
| 1장 선택, 나머지 소멸 |
| 빌드 아키타입 시너지가 가중치에 반영(이미 든 빌드 계열이 더 자주 등장) |

### 4.2 희귀도 가중치

| 등급 | 가중치 `[balance]` | 성격 |
|---|---|---|
| Common | 60% | 안정적 행동 변화 |
| Rare | 30% | 강한 시너지/조건부 |
| Exotic | 10% | 빌드를 통째로 바꾸는 핵심 |

### 4.3 자산·키워드 무작위

- Asset 노드 회수 시 내용물(Credits/키워드 칩/단서)이 무작위.
- 수집형 키워드 칩(`EXPLORER_INTERACTION §7.3`)도 런마다 다르게 등장.

## 5. Generation RNG — 런 시작 무작위

### 5.1 레벨 시드

내부 배치를 시드로 변주하되, **구역 성격은 고정**한다. (`LEVEL_GENERATION_DESIGN.md §8`)
목표는 항상 도달 가능(불운으로 클리어 불가한 맵은 생성 금지).

### 5.2 미션 변수 (Boon / Bane)

런 시작 시 무작위 변수 1~2개가 붙어 같은 미션도 다르게 만든다.

| 분류 | 예시 |
|---|---|
| Boon(유리) | "Indexer 1기 결근", "temp 폴더 2개", "키워드 칩 1개 선지급" |
| Bane(불리) | "Firewall 추가 배치", "제한 턴 -1", "Backup Repairer 강화" |
| 양날 | "고가치 자산 추가 + 보안 레벨 +1" |

> 변수는 **출격 준비 화면에서 공개**된다(`LOBBY_MISSION_DESIGN §8`). 모르고 들어가지 않는다.

### 5.3 보안 구성 변동

보안 레벨 내에서 어떤 모듈이 어디에 배치될지는 일부 무작위. 단 `SECURITY_AI §5`의
"목표 근처일수록 조밀" 규칙은 항상 유지.

---

## 6. Reaction RNG — 보안 턴 무작위

보안의 예측 불가능성을 위한 **작은** 무작위. 큰 결정은 결정론적(Trace 단계)으로 유지.

| 무작위 요소 | 변동폭 |
|---|---|
| Guard 순찰 분기 선택 | 갈림길에서 경로 택일 |
| 감시 대상 노드 선택 | 동급 후보 중 택일 |
| 방해(Interrupt) 발생 | 감시 중 작업에 확률적 발생 |

> 보안의 "다음 행동 예고"는 유지된다. 무작위는 *예고된 선택지 중 어느 것이냐*에만 적용되어,
> 플레이어가 완전히 허를 찔리지 않게 한다.

## 7. 변동성 완화 (Variance Control)

운에 휘둘리기 싫은 플레이어를 위한 **실력 기반 완화 수단**. 운빨과 실력빨이 공존한다.

| 수단 | 효과 |
|---|---|
| 확률 사전 공개 | 모든 판정·점프 확률을 커밋 전 표시 |
| Analyst 빌드 | 위험 사전 식별, 일부 판정 변동폭 축소 |
| Lock-in 도구 | 다음 판정을 "성공 확정"으로 고정(소비) |
| Reroll 도구 | 드래프트/변수 1회 다시 뽑기(소비) |
| 정보 수집 | 키워드·Scan으로 함정·목표를 미리 파악 |

> 설계 의도: 운을 **줄이는 것 자체가 빌드 선택지**다. "운빨 빌드 vs 안정 빌드"가 모두 유효해야 한다.

## 8. 불운 보호 (Bad-Luck Protection)

연속 불운이 곧 패배가 되지 않도록 바닥을 둔다.

| 장치 | 규칙 `[balance]` |
|---|---|
| 펌블 천장(pity) | 같은 액션 펌블 2연속 시 다음 판정 펌블 확률 0 |
| 드래프트 보장 | 일정 횟수 내 Rare 이상 1장 보장 |
| 단일 판정 비치명 | 한 번의 실패로 즉시 런 종료되지 않음(Trace↑까지) |
| 목표 도달 보장 | 클리어 불가 레이아웃 생성 금지 |
| Heat 감소 경로 | 불운 누적이 영구 데스 스파이럴이 되지 않게 |

## 9. 시드 & 데일리 런 (선택 기능)

| 기능 | 설명 |
|---|---|
| 시드 공유 | 동일 시드 → 동일 Generation RNG. 챌린지·공략 공유 가능 |
| 데일리 런 | 모두에게 같은 시드의 하루 한정 런 → 순위 비교 |

> 구현은 후순위. 단, RNG를 **단일 시드 스트림**에서 끌어오도록 초기부터 설계하면 추후 무료로 얻는다.

## 10. 무작위성 vs 결정론 — 한눈 정리

| 영역 | 방식 | 이유 |
|---|---|---|
| 파일 작업 진행률 | 결정론 | 계획의 뼈대, 운으로 흔들리면 안 됨 |
| Trace 단계 전환 | 결정론 | 보안 반응은 예측 가능해야 |
| 고위험 액션 | 판정(공개) | 도박의 긴장 |
| 미검증 점프 | 판정(공개) | 탐험의 도박 |
| 레벨/보상/변수 | 무작위 | 런 다양성 |
| 보안 미시 선택 | 소폭 무작위 | 예측 불가의 양념 |
