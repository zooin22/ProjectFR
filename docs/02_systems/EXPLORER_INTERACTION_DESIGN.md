# ProjectFR - 탐색기 인터랙션 기획서 (Explorer Interaction)

> Status: design — 조작 모델 명세 (선택후명령 / 우클릭 / 드래그 / 탐색기 기능의 전술화)
> Read with: `COMBAT_SYSTEM_DESIGN.md`, `UI_UX_DESIGN.md`, `GAME_CONCEPT.md`
> Owns: 입력 모델, 우클릭 컨텍스트 메뉴, 드래그앤드롭, 보기/정렬/검색/속성/휴지통, 클립보드·볼주머니

---

## 1. 설계 선언

> 전투 시스템을 따로 만들지 않는다. **윈도우 탐색기 조작 자체가 전투 시스템이다.**

이 문서는 "탐색기에서 누구나 아는 조작"을 어떻게 **전술 선택지**로 바꾸는지를 정의한다.
모든 입력은 단순 UI 기능이 아니라, 시간·Trace·보안 반응이라는 비용과 위험을 갖는다.

핵심 원칙:

```text
정밀 조작 = 우클릭 컨텍스트 메뉴
빠른 조작 = 하단 퀵바 (클릭)
빌드 확장 = 메뉴 항목 추가 또는 변형
```

### 1.1 포인터 전용 (No-Keyboard) 원칙

전투는 **계획형 턴제**다. 한 손을 키보드에 올려두고 타이핑하는 흐름은 턴제의 "느긋하게
계획하고 한 번에 실행하는" 리듬과 맞지 않는다. 따라서 ProjectFR의 모든 조작은
**마우스/포인터만으로 완결**되도록 설계한다.

| 규칙 | 내용 |
|---|---|
| 텍스트 입력 금지 | 검색어·경로·파일명을 직접 타이핑하지 않는다 |
| 키보드는 보조 전용 | 단축키는 "있어도 되는" 편의이지 필수가 아니다 |
| 모든 행동은 클릭 가능 | 좌클릭 / 우클릭 / 더블클릭 / 드래그 / 호버로 완결 |
| 자유 입력 → 선택지로 | "키워드 타이핑"은 "수집한 키워드 칩 클릭"으로 대체 |

> 이 원칙은 본 문서의 §7(검색)·§8(주소창)·§9(속성/이름)에 직접 영향을 준다.
> 키보드 의존 기능은 모두 클릭 기반 대안으로 재설계되었다.

## 2. 기본 조작 모델 — 선택 후 명령

플레이어는 대상을 선택하고 가능한 명령을 큐에 적재한다.
(실행 흐름은 `COMBAT_SYSTEM_DESIGN.md` §2 명령 큐)

| 선택 대상 | 가능한 명령 |
|---|---|
| 파일 | Copy / Delete / Compress / Properties / Rewrite Log / Restore / Mark Important |
| 폴더 | Open / Scan / Show Hidden / Permission Override |
| 보안 모듈 | Observe / Stun / Inject / Decoy 유도 / Quarantine 대상 지정 |
| 빈 공간 | Decoy 생성 / Paste / 임시 경로 생성 |

### 2.1 선택 상태 피드백

- 선택 노드는 탐색기 선택 박스처럼 강조.
- 커서 위치와 선택 노드가 카드 텍스트/색상에 반영(Step 3 구현).
- 선택 시 우측 인스펙터에 상세 정보·작업 진행률·기억 단서 표시.

## 3. 우클릭 컨텍스트 메뉴 — 핵심 전투 인터페이스

우클릭 메뉴는 보조 UI가 아니라 **주력 명령 패널**이다. 빌드는 이 메뉴 항목을 늘리거나 변형한다.

### 3.1 대상별 메뉴

**파일 우클릭**
```text
열기 · 복사 · 이동 · 삭제 · 이름 바꾸기 · 압축 · 속성
로그 위조 · 권한 분석 · 미끼로 복제 · 복원 · 보호 표시
```

**보안 모듈 우클릭**
```text
관찰 · 기절 · 우회 · 감염 · 미끼로 유도 · 격리 · 제거
```

**폴더 우클릭**
```text
진입 · 스캔 · 숨김 파일 표시 · 방화벽 우회 · 임시 경로 생성 · 감시 해제
```

### 3.2 메뉴 항목의 동적 활성화

항목은 상황에 따라 활성/비활성된다. 예:

| 조건 | 활성화되는 항목 |
|---|---|
| 선택 노드가 클립보드에 있음 | Store in Pouch |
| 선택 노드가 볼주머니에 있음 | Restore to Clipboard |
| 폴더가 Forced Lock 상태 | Permission Override |
| 파일이 휴지통에 있음 | Restore |
| 파일이 ghost 상태 | Show Hidden / Permission Override(은닉 해제) |

> 구현 메모(Step 5): 중앙 전술 공간 우클릭 시 `ExplorerContextMenu` 팝업, 선택 노드 기준으로
> Move Cursor / Open / Properties / Copy / Compress / Rewrite Log / Delete 등을 큐 적재.

## 4. 드래그 앤 드롭

직관적이지만 게임적으로는 비용·위험을 갖는다.

| 행동 | 효과 | 위험 |
|---|---|---|
| 파일 → 다른 폴더 | 빠른 이동 | 이동 로그 생성 |
| 파일 → zip | 은닉/압축 | 압축 시간 발생 |
| 보안 파일 드래그 | 강제 이동 | Trace 크게↑ |
| 감시 중 파일 드래그 | 작업 시도 | Guard 반응 |
| 휴지통 파일 → 원래 폴더 | 복원 | Backup/Indexer 반응 |
| 회수 파일 → extraction zone | 운반 | — |

> 구현 메모(Step 5): 드래그 시작/드롭 대상이 폴더/아카이브면 `Move`/`Compress` 큐 적재,
> 드롭 완료 시 `BattleDungeon.MoveNode(...)`로 컨테이너 재배치.

## 5. 보기 모드 (View Mode)

보기 모드는 옵션이 아니라 **정보 vs 전술 가독성**의 트레이드오프다.

| 보기 모드 | 장점 | 단점 |
|---|---|---|
| 큰 아이콘 | 파일/보안 위치 파악 쉬움 (기본 전술 뷰) | 세부 정보 부족 |
| 자세히 | 크기·수정일·권한·위험도 확인 | 전술 배치 가독성↓ |
| 목록 | 많은 파일 빠르게 확인 | 위험 표현 약함 |
| 트리 | 전체 경로·우회 루트 파악 | 현재 폴더 전투 정보 약함 |
| 숨김 표시 | 숨겨진 목표 발견 | Trace↑ 또는 Indexer 반응 |

> 기본 전술 공간은 "큰 아이콘 보기"다. 필드가 아니라 *탐색기 자체가 보드*가 되도록.

## 6. 정렬·필터링

정렬도 전술이다. 단, 필터링은 분석 행동으로 간주되어 Trace를 올릴 수 있다.

| 정렬 기준 | 게임 효과 |
|---|---|
| 이름순 | 목표 파일 탐색 쉬움 |
| 수정 날짜순 | 최근 변경 로그·함정 추적 |
| 크기순 | 대용량 중요 데이터 식별 |
| 형식순 | `.log/.jpg/.txt/.zip/.lnk` 분리 |
| 보안 등급순 | 위험 파일 우선 확인 |
| 오래된 항목순 | 삭제 대상 기억 파일 발견 |

## 7. 검색 (Search) — 타이핑 없는 클릭 기반

검색은 강력하지만 시끄러운 액션이다. **키워드를 타이핑하지 않는다.**
검색은 세 가지 클릭 메커니즘으로 분리된다.

### 7.1 Search Sweep (키워드 없는 스윕)

검색 버튼 클릭 = 현재 폴더/선택 노드 기준으로 **목표 방향을 탐지**한다.
키워드 입력 없이, 미션 목표를 기준으로 근접도 단서를 돌려준다.

| 결과 | 의미 |
|---|---|
| `exact hit` | 목표가 현재 컨테이너 안에 있음 |
| `nearby` | 같은 구역 인접 폴더에 있음 |
| `deeper hint` | 더 깊은 하위 경로에 있음 |
| `no hit` | 이 방향엔 없음 |

> 구현 메모(Step 5에 이미 존재): Search 결과를 `exact hit / nearby / deeper hint / no hit`로
> 분류. 이 분류를 **메인 검색 메커니즘**으로 승격하면 타이핑 없이 그대로 동작한다.

### 7.2 Filter Chips (클릭 토글)

타이핑 대신 미리 정의된 **필터 칩**을 클릭해 켜고 끈다.

```text
[.jpg] [.log] [.zip] [.lnk]   [오래된순] [숨김] [위험] [기억]
```

- 칩은 복수 토글 가능(예: `.jpg` + `오래된순` → 오래된 사진 후보만).
- 칩 사용은 §6 정렬·필터와 동일하게 분석 행동 → Trace 소폭↑.

### 7.3 Known Keywords (수집형 키워드 칩)

특정 대상을 콕 집어 찾는 정밀 검색은 **수집한 키워드 칩**을 클릭해서 한다.
키워드는 타이핑 자원이 아니라 **수집 자원**이다.

| 규칙 |
|---|
| 단서(로그·메모·브리핑)를 회수하면 키워드 칩이 해금됨 (예: `zone17`, `family`, `home`) |
| 해금된 키워드는 검색 패널에 클릭 가능한 칩으로 누적 표시 |
| 칩을 클릭하면 해당 키워드로 정밀 검색(정확한 후보 강조) |
| 보유하지 않은 키워드는 검색 불가 → "정보를 먼저 찾아야 한다"는 압박 |

> 로그라이크 시너지: 키워드 칩은 런마다 다르게 등장하는 **수집형 단서**가 된다.
> Analyst 빌드는 시작 시 키워드 1~2개를 미리 보유할 수 있다.

### 7.4 공통 위험

| 효과 | 위험 |
|---|---|
| 목표 후보 표시 | Indexer Scout 활성화 |
| 숨김 파일 일부 탐지 | Trace↑, 검색 로그 생성 |
| 관련 로그 발견 | 검색 자체가 Active Scan을 유발 |

> 과거 "잘못된 키워드 → honeypot에 낚임"의 도박성은 §8의 ShortcutLink 클릭(무키보드 점프)으로
> 이전한다. 검색은 더 이상 오타로 함정에 빠지지 않는다.

## 8. 내비게이션 — 클릭 기반 (주소창 타이핑 대체)

경로를 직접 타이핑하지 않는다. 빠른 이동은 세 가지 **클릭 수단**으로 한다.

### 8.1 브레드크럼 (Breadcrumb)

주소창은 입력란이 아니라 **클릭 가능한 경로 조각**이다.

```text
root  ›  cache  ›  archive  ›  zone_17
 ↑클릭하면 그 폴더로 점프(상위 이동)
```

- 각 조각 클릭 → 해당 폴더로 즉시 이동(안전, 이미 거쳐온 경로).
- 상위 이동도 감시 반응·Trace를 유발할 수 있음(`HandleFolderNavigation`).

### 8.2 Quick Access 북마크

좌측 패널의 빠른 접근 항목을 클릭해 자주 가는 구역으로 점프.

| 항목 | 의미 |
|---|---|
| 고정 북마크 | temp/, recycle_bin/ 등 상시 핀 |
| 런 중 핀 | 플레이어가 우클릭→핀으로 추가(클릭) |

### 8.3 ShortcutLink 점프 (도박성 이동)

과거 "주소창 오입력 → honeypot" 도박은 여기로 이전된다.
`.lnk` 노드(ShortcutLink)를 클릭해 점프하는 행위가 **운 요소**가 된다.

| 바로가기 상태 | 클릭 결과 |
|---|---|
| 검증됨(복구 완료) | 안전한 우회 점프, 빠른 탈출로 |
| 미검증/손상 | **도박**: 성공 시 깊은 구역 진입, 실패 시 honeypot/Trace↑ |

- 미검증 점프는 **확률이 사전 표시**된다(§커밋 전 공개, `RANDOMNESS_DESIGN.md` 참조).
- Repair로 바로가기를 검증하면 도박이 안전 점프로 바뀐다.

> 정리: 안전 이동은 브레드크럼/북마크 클릭, 위험·고보상 이동은 ShortcutLink 도박 클릭.
> 둘 다 타이핑이 없다.

## 9. 속성 창 (Properties)

속성은 핵심 조작 대상이다.

| 속성 | 전술 활용 |
|---|---|
| 숨김 | 목표 파일을 숨겨 감시 회피 |
| 시스템 파일 | decoy를 시스템 파일로 위장 |
| 읽기 전용 | 로그를 읽기 전용으로 만들어 수정 차단 |
| 압축됨 | 위험 파일을 압축 상태로 만들어 스캔 회피 |
| 소유자 | 소유자 변경으로 접근 허가 위조 |
| 중요 표시 | 오래된 사진을 CleanSweep 우선순위에서 제외 |

### 9.1 이름 바꾸기 (Rename) — 프리셋 선택

이름 바꾸기는 전술이지만 **파일명을 타이핑하지 않는다.** 대신 미리 정의된
**위장 프리셋**을 클릭해서 적용한다.

| 프리셋 | 효과 |
|---|---|
| 시스템 파일처럼 | 보안 우선순위에서 일반 파일로 위장 |
| 미끼 이름으로 | decoy로 인식되게 변경 |
| 무해 확장자로 | `.log` → `.txt` 등으로 위험도 표시 완화 |
| 원래대로 | 위장 해제 |

> 자유 입력이 필요한 연출(예: 플레이어가 직접 이름을 짓는 재미)은 의도적으로 배제했다.
> 전술 효과는 "무엇으로 위장하느냐"이지 "어떤 글자냐"가 아니기 때문이다.

## 10. 휴지통 (Recycle Bin)

삭제는 끝이 아니라 휴지통으로의 이동이다. 휴지통은 "삭제된 존재들의 유배지".

| 행동 | 효과 | 위험 |
|---|---|---|
| Delete | 휴지통으로 이동 | 흔적 남음 |
| Shift + Delete | 완전 삭제 | Trace 크게↑ |
| 휴지통 비우기 | 삭제 확정 | 보안 경고 |
| Restore | 삭제 취소 | Backup Repairer와 상호작용 |

서사적 의미: 휴지통에서 깨진 사진·세이브·메일을 복원하는 것이 주인공의 첫 동기다.

## 11. 클립보드 & 볼주머니 캐시 — 햄스터 고유 자원

### 11.1 클립보드 (Clipboard)

| 규칙 |
|---|
| 복사한 파일이 들어감 |
| 제한 용량 보유 |
| 가득 차면 Paste 전까지 추가 복사 제한 |

### 11.2 볼주머니 캐시 (Cheek Pouch Cache)

주인공이 햄스터인 점을 살린 특수 은닉 슬롯.

| 규칙 |
|---|
| `PouchMaxFileSize` 이하 작은 파일만 저장 가능 |
| oversized 파일은 저장 차단 + 차단 사유 표시 |
| 저장 파일은 `ghost` 상태 → Trace 소폭 감소(지속) |
| ghost 파일은 Indexer/AI Monitor 일부 감시·Search 정확도에서 제외 |

### 11.3 ghost 해제

`Show Hidden` / `Permission Override`로 pouch-hidden 파일을 `Pouch Exposed`로 바꿔
은닉을 일부 해제. 보안 측 대응 카드로 기능.

### 11.4 클립보드↔볼주머니 흐름

```text
파일 Copy → Clipboard
Clipboard 항목 → [Store in Pouch] → ghost화 (작으면)
Pouch 항목 → [Restore to Clipboard] → 다시 운반 가능
```

(전용 창 UI는 `MULTI_WINDOW_EXPANSION.md` Clipboard Window, `UI_UX_DESIGN.md` §멀티윈도우)

### 11.5 관련 빌드 (요약, 상세는 PROGRESSION_ECONOMY)

| 빌드 요소 | 효과 |

## 12. 작업 관리자 (Task Manager) — 후반 시스템

보안 모듈은 캐릭터로 보이지만 시스템적으로는 프로세스다.
작업 관리자는 이 "프로세스 레이어"를 직접 조작하는 고급 인터페이스.

대상 프로세스 예: `AntivirusCore.exe`, `IndexerService.exe`, `BackupRepairer.exe`,
`FirewallGate.sys`, `AI_Monitor.dll`, `CleanSweepAgent.exe`, `OPTIMA_Service.sys`

| 조작 | 효과 |
|---|---|
| 프로세스 종료 | 모듈 제거(고위험, Trace 급등) |
| 일시 중지 | 모듈 Stun과 유사 |
| 우선순위 낮추기 | 모듈 반응 속도↓ |
| 권한 탈취 | 일시적 관리자 권한 획득 |
| 가짜 프로세스 위장 | 자신을 정상 프로세스로 은폐 |
| 감염 코드 주입 | 모듈을 아군처럼 동작시킴 |

## 13. 입력 ↔ 명령 매핑 요약 (포인터 전용)

모든 명령은 마우스만으로 완결된다. 키보드는 보조 단축키일 뿐 필수가 아니다.

| 입력 | 결과 |
|---|---|
| 좌클릭 | 선택 |
| 더블클릭 | 폴더 열기 / 파일 Access |
| 우클릭 | 컨텍스트 메뉴 |
| 드래그 | Move/Compress 큐 적재 |
| 호버 | 노드 상태·배지 툴팁 |
| 하단 퀵바 클릭 | 자주 쓰는 명령 즉시 큐 적재 |
| 브레드크럼/북마크 클릭 | 안전한 폴더 점프 |
| ShortcutLink 클릭 | 도박성 점프(확률 사전 표시) |
| 검색 버튼 클릭 | Search Sweep(키워드 없는 방향 탐지) |
| 필터 칩 / 키워드 칩 클릭 | 필터링 / 정밀 검색 |

> 타이핑이 필요한 입력은 없다. 모든 "자유 입력"은 클릭 가능한 선택지로 환원되었다.
