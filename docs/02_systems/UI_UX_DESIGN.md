> Status: design — 화면 레이아웃 · 멀티윈도우 · 상태 가시화 명세

---

## 1. 설계 선언

ProjectFR의 화면은 게임 HUD가 아니라 **"작전용 데스크톱"** 처럼 보여야 한다.

> 전통적 전투 HUD가 아니라, 데스크톱 위에서 여러 탐색기 창을 운영하는 침투 시뮬레이션.

핵심 원칙:

```text
탐색기 배경 위에서 싸우는 것이 아니라, 탐색기 자체가 전술 보드가 되어야 한다.
```

플레이어가 "나는 던전을 도는 게 아니라, 위험한 컴퓨터 안에서 실제로 파일을 조작하고 있다"고
느끼게 하는 것이 UI의 최종 목표다.

## 2. 기본 화면 레이아웃

실제 윈도우 탐색기와 최대한 유사한 4분할 + 상/하단 바.

```text
┌───────────────────────────────────────────────────────────┐
│ [리본 메뉴]                                                  │
│ [주소창]  /cache/archive/zone_17/        [검색창]            │
├──────────┬─────────────────────────────────┬───────────────┤
│ 좌측      │ 중앙: 큰 아이콘 보기 전술 공간      │ 우측: 인스펙터   │
│ 폴더 트리 │                                  │ - 미션 목표      │
│ 빠른 접근 │  [📁]  [📄]  [🔒guard]            │ - 선택 정보      │
│ 드라이브  │  [📄objective] [🧀cursor]         │ - 보안 반응 예고  │
│ 봉쇄/격리 │  [📄] [decoy] [📦zip]             │ - 작업 진행률     │
│ 상태      │                                  │ - 기억/단서      │
│          │  상단 요약: Cursor 상태 · 보안 배치 │ - Trace 원인     │
├──────────┴─────────────────────────────────┴───────────────┤
│ [퀵바: Access Copy Delete Restore Compress Stun RewriteLog] │
│ [작업 큐 1..N]   [Clipboard]  [Pouch]   AP 3/3  [명령 실행]  │
└───────────────────────────────────────────────────────────┘
```

| 영역 | 담당 |
|---|---|
| 상단 | 리본 메뉴 · 주소창 · 검색창 |
| 좌측 | 폴더 트리 · 빠른 접근 · 드라이브 · 봉쇄/격리 상태 |
| 중앙 | 큰 아이콘 보기 기반 파일/폴더/보안/커서 전술 공간 |
| 우측 | 선택 정보 · 미션 목표 · 보안 반응 · 작업 진행률 · 기억 단서 |
| 하단 | 명령 퀵바 · 작업 큐 · 클립보드 · 볼주머니 · AP · 실행 버튼 |

## 3. 중앙 전술 공간

"필드"가 아니라 **큰 아이콘 보기 탐색기**처럼 보여야 한다.

원칙:
- 파일·폴더는 격자 정렬.
- 보안 캐릭터는 파일 앞/옆 슬롯에 배치.
- 플레이어는 해커 햄스터 + 커서 표시로 표현.
- 이동 경로는 **명령 예약 중에만** 표시(상시 표시 X — 탐색기 감각 유지).
- 시야 범위는 적 선택/위험 예측 시에만 표시.
- 목표 파일은 탐색기 선택 박스처럼 강조.
- 삭제 대기열 파일은 흐려지거나 점멸.
- 복원 가능 파일은 휴지통/깨진 아이콘으로 표시.

> 구현 메모(Step 3): 중앙에 `ExplorerFieldList`(큰 아이콘 카드), `FileListTree` 숨김,
> `CursorStatusLabel`/`FieldSecurityLabel`로 상단 요약 표시. 선택/더블클릭으로 노드 선택·폴더 열기.

## 4. 우측 인스펙터 패널

선택 대상과 작전 상황을 읽는 곳.

| 표시 항목 |
|---|
| 현재 미션 목표 |
| 선택된 파일/보안 모듈 정보 |
| 현재 Trace 원인 |
| 다음 보안 반응 예고 |
| 진행 중인 파일 작업(%) |
| CleanSweep 삭제 대기열 |
| 해당 파일의 기억/스토리 단서 |
| 상태: Access(Locked/Overridden/Open), Pouch(Hidden/Exposed/Safe), Oversize 등 |

> oversized 파일은 인스펙터 힌트에서 pouch 한계값 대비 초과 크기를 직접 안내.

## 5. 하단 패널

```text
작업 큐
1. Scanner 기절
2. family_photo.jpg 복원
3. deletion_report.log 위조
```

포함: 명령 퀵바 · AP/명령 슬롯 · 작업 큐 · 클립보드 내용 · 볼주머니 내용 ·
실행 버튼 · 턴 종료 버튼.

> 구현 메모(Step 4): 액션 클릭 시 즉시 실행 대신 Command Queue 적재, `Execute Queue`/`Clear Queue`,
> 최근 Activity 패널에 진행 중 작업 %/상태 표시.

## 6. 멀티윈도우 — 데스크톱 오버레이

(단계 확장 상세는 `MULTI_WINDOW_EXPANSION.md`)

### 6.1 표현 방향

- 메인 BattleScene은 전체화면이 아니라 **데스크톱 위의 메인 침투 창**처럼 보이게 한다.
- 서브창은 떠 있는 보조 패널이 아니라, **제목바·닫기 버튼이 있는 독립 내부 창**으로 띄운다.
- 포커스된 창이 시각적으로 명확해야 한다(한 번에 직접 명령 가능한 창은 하나).

### 6.2 창 타입

| 창 | 역할 | 상태 |
|---|---|---|
| Main Infiltration | 기본 전술 공간(항상 존재) | 구현 |
| Clipboard | 클립보드·볼주머니 관리 | 구현(Phase A) |
| Temp | 임시 경유·은닉 | 미구현 |
| Log Viewer | 로그 조작 전용 | 미구현 |
| Backup | 복원 대응 | 미구현 |
| Archive | 압축 내부 관리 | 미구현 |
| Recycle Bin | 삭제 파일 복원 | 미구현 |

### 6.3 Clipboard Window (구현 완료)

```text
┌─ Clipboard ───────────────[_][x]─┐
│ Clipboard:                        │
│  • family_photo.jpg  (ghost)      │
│ Cheek Pouch Cache:                │
│  • diary.txt  (safe)              │
│ Focus/Bind: /cache/archive        │
│ [Store in Pouch] [Restore to CB]  │
└───────────────────────────────────┘
```

규칙: 열기 시 Trace +1. 열려 있으면 버튼으로 포커스 복귀, Close로 닫기.
선택 노드가 clipboard에 있으면 Store in Pouch 활성, pouch에 있으면 Restore 활성.

### 6.4 멀티윈도우 비용 (UX ↔ 게임 규칙)

| 행동 | 비용 |
|---|---|
| 창 열기 | Trace +1 |
| Log Viewer 유지 중 Search | Trace +1 추가 |
| 여러 창 동시 Active Operation | Alert 가속 |

UI는 이 비용을 **상단 요약**에 노출: `Windows N · Focus X`.

## 7. 상태 가시화 — 배지 시스템

보안 상태효과(`SECURITY_AI_DESIGN §5`)를 탐색기 전역에서 즉시 읽히게 한다.

### 7.1 텍스트 배지

| 배지 | 의미 |
|---|---|
| `[WATCH]` | 감시 중(SightRange 내) |
| `[LOCK]` | 폴더 봉쇄 |
| `[OVERRIDE nT]` | 권한 우회 잔여 턴 |
| `[TRACKED nT]` | 표식 잔여 턴 |
| `[PRESSURE nT]` | 스캔 압박 잔여 턴 |
| `[SWEEP nT]` | 삭제 카운트다운 |

### 7.2 표시 위치 (일관성 규칙)

| 위치 | 표시 |
|---|---|
| 파일 트리 status 컬럼 | 대표 배지 아이콘 1개 |
| 중앙 카드 보조 텍스트 | compact badge summary |
| 툴팁 | 전체 배지 + 설명 |
| 인스펙터 | 전체 상태 + 잔여 턴 + 해제 방법 |
| 상단 요약 | Trace 단계 · Windows N · Focus X |

> 구현 메모(Step 7): placeholder SVG 배지를 BattleScene에 연결, 트리 status 컬럼 + 카드
> 보조 텍스트에 적용. 상단/`status` 콘솔 요약에도 윈도우 상태 포함.

## 8. UX 핵심 가이드라인

| 원칙 | 설명 |
|---|---|
| 진짜 탐색기처럼 | 친숙한 OS 조작을 그대로 가져와 학습 비용↓ |
| 위험은 항상 예고 | 보안 다음 행동을 예고해 계획형 턴제 성립 |
| 상태는 한눈에 | 배지로 위험을 즉시 읽히게(텍스트 의존↓) |
| 정보 vs 가독성 | 보기 모드로 트레이드오프를 플레이어가 선택 |
| 비용 있는 편의 | 멀티윈도우·검색 등 강력한 편의엔 Trace 비용 |
| 연출은 실시간처럼 | 턴 실행은 짧은 애니메이션으로 흐르게 |

## 9. UI 단계 로드맵

### Stage 1 (현재 지향)
- 메인 침투 창이 화면 대부분, 배경/여백으로 "데스크톱 위 앱 창" 느낌.
- Clipboard 같은 보조창은 제목바+닫기 있는 독립 내부 창.
- 창 겹침 제한적 허용, 포커스 창 명확.

### Stage 2
- 드래그 가능한 내부 창.
- 창마다 제목바/닫기/핀/고정 경로.
- Temp/Log Viewer/Backup 병렬 운용.
- 최종: "전술 그리드 단일 화면"보다 "작전 데스크톱에서 여러 창 관리" 쪽.
